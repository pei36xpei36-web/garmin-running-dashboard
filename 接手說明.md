# 跑步儀表板專案 — 接手說明

> 給接手的人（或 Claude Code）看的說明。最後更新：2026-07-02。

---

## 一句話說明

個人跑步資料儀表板，已從「純本機」升級成「跨裝置雲端版」：
從 Garmin 抓資料 → 存本機 SQLite + 同步到 **Supabase Postgres 雲端** → 儀表板部署在
**Streamlit Community Cloud**，手機／辦公室／家裡打開同一個網址、輸入密碼就能看。

## 架構

```
[任一台 PC] python fetch_data.py --update
      │  (1) 寫本機 garmin_data.db   (2) 同步 upsert 到 Supabase
      ▼
[Supabase Postgres]  雲端單一資料來源（3 張表：activities / daily_wellness / hr_zones）
      ▲  讀取（連線字串放在 Streamlit Secrets）
[Streamlit Community Cloud]  部署 dashboard.py
      ▲  https 網址 + 密碼登入
[手機 / 辦公室 / 家裡 瀏覽器]
```

- 資料來源由 `db.py` 決定：**有 `SUPABASE_DB_URL` 就走雲端，沒有就走本機 `garmin_data.db`**。同一份程式兩邊都能跑。
- Supabase 專案 id：`gdiqywcvgxzbyqldbnjw`（region ap-southeast-1）。

## 檔案說明

| 檔案 | 用途 |
|------|------|
| `fetch_data.py` | 從 Garmin 抓資料，寫本機 SQLite，並在有設定時同步到 Supabase。支援 `--update`、`--days N` |
| `db.py` | 資料存取層：偵測 `SUPABASE_DB_URL` → 連雲端，否則連本機 SQLite |
| `dashboard.py` | Streamlit 儀表板 + 密碼登入閘門（`require_login`） |
| `sync_to_cloud.py` | 把本機 SQLite 現有資料一次推上 Supabase（不需重連 Garmin） |
| `requirements.txt` | 套件清單 |
| `.env` | 本機密鑰（Garmin 帳密、`SUPABASE_DB_URL`、`APP_PASSWORD`）**不進 git** |
| `.env.example` / `.streamlit/secrets.toml.example` | 範本 |

## 密鑰（放哪裡）

- **本機**：`.env`
  - `GARMIN_EMAIL` / `GARMIN_PASSWORD`：抓資料用
  - `SUPABASE_DB_URL`：Supabase「Session pooler」連線字串（port 5432、host 含 `aws-1-ap-southeast-1.pooler.supabase.com`）
  - `APP_PASSWORD`：儀表板登入密碼
- **Streamlit Cloud**：專案 Settings → Secrets，放 `SUPABASE_DB_URL` 與 `APP_PASSWORD`（TOML 格式，值要加雙引號）。
- **絕不進 git**：`.env`、`.garmin_tokens/`、`garmin_data.db`、`.streamlit/secrets.toml`（已在 .gitignore）。

## 日常使用

- **更新資料**（任一台 PC，需已設好 `.env`）：
  ```
  venv\Scripts\python fetch_data.py --update
  ```
  → 寫本機並同步到 Supabase → 手機／各裝置重新整理網址即見新資料（**不需 git**）。
- **本機開儀表板**：`venv\Scripts\streamlit run dashboard.py`
- **改程式／圖表後重新部署**：`git push` → Streamlit Cloud 自動重新部署。

## 首次部署到 Streamlit Community Cloud（一次性）

1. 程式已推到 GitHub private repo（見 `git remote -v`）。
2. 到 https://share.streamlit.io ，用 GitHub 登入。
3. New app → 選這個 repo → 分支 `main` / 主檔 `dashboard.py` → Deploy。
4. App 的 Settings → Secrets 貼上（值從自己的 `.env` 複製）：
   ```toml
   SUPABASE_DB_URL = "postgresql://postgres.gdiqywcvgxzbyqldbnjw:你的密碼@aws-1-ap-southeast-1.pooler.supabase.com:5432/postgres"
   APP_PASSWORD = "你的登入密碼"
   ```
5. 部署完成會給一個網址，手機／辦公室打開輸入 `APP_PASSWORD` 就能看。

## 在不同電腦上抓資料

`venv/` 不能跨電腦搬，需在該電腦重建：
```
python -m venv venv
venv\Scripts\python -m pip install -r requirements.txt
```
再把 `.env`（含 Garmin 帳密 + `SUPABASE_DB_URL`）建好，就能在那台 `fetch_data.py --update` 更新雲端。
（`.env` 不在 git 裡，換電腦要自己照 `.env.example` 重建一份。）

- **`review.py`：週/月復盤產生器**，數字全來自實際資料、主觀欄位留白，寫進 Obsidian vault：
  - `python review.py --week`（上一個完整週）或 `--week 2026-06-25`
  - `python review.py --month 2026-06`
  - 需在 `.env` 設 `OBSIDIAN_VAULT`（本機 vault 路徑，例 `D:\Secondbrain`）。
  - 週復盤寫到 `Myself/馬拉松/週復盤/`，月報寫到 `Myself/馬拉松/`，自動對照 27 週訓練計畫的週目標。
